64 // A `Texture_sprite` is a `Sprite` that can be rendered by copying
65 // a texture. Instead of specifying how to render themselves directly,
66 // derived classes must specify how to get a `Texture` representing
67 // the sprite. The dimensions of the sprite are the dimensions of the
68 // resulting texture. The return type of `Texture const&` means that
69 // `get_texture_` cannot just create and return a texture, but must
70 // store it somewhere. This is because `Texture`s will usually be
71 // cached. (Otherwise, you wouldn't use a `Texture_sprite`.)
72 class Texture_sprite : public Sprite
73 {
74 public:
75  Dimensions dimensions() const override;
76 
77 private:
78  void render(detail::Renderer&, Position, Transform const&) const override;
79  void prepare(detail::Renderer const&) const override;
80 
81  virtual Texture const& get_texture_() const = 0;
82 };

84 // A `Render_sprite` works by allowing its derived classes to render
85 // themselves onto an `SDL_Surface`, which it creates. It then converts
86 // that surface to a `Texture`, which it caches.
